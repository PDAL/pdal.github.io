Docker¶

Author:Howard Butler
Contact:howard@hobu.co
Date:12/02/2015

Introduction¶

It’s a giant pain to build everything yourself. Like the Vagrant configuration, a PDAL build based on Docker is also available. This document describes how to use it to operate on data.

See also

The What is Docker document describes in more detail what exactly Docker is. Think of it as a virtualization platform that doesn’t have to be “built” every time from scratch like Vagrant.

Prerequisites¶

Install Docker.

  • Windows
  • OSX
  • Linux

Note

This tutorial will assume you are running on Windows, but the same commands should work in OSX or Linux too.

Data¶

Copy some LAS data to the directory you’re running the shell

What you get¶

The :ref:`docker`_ configuration that PDAL provides contains nearly every possible feature except for Oracle Point Cloud support. Things it includes are:

  • filters.hexbin
  • filters.reprojection
  • readers.nitf and writers.nitf
  • LASzip support in readers.las and writers.las
  • Interpolation with Points2Grid using writers.p2g
  • Python manipulation using filters.predicate and filters.programmable
  • PCL support via filters.pclblock, readers.pcd, writers.pcd, ground command, and pcl command
  • writers.geowave

Head to Pipeline for more information on using PDAL pipelines. Two pipelines are provided in /home/vagrant that are used to load the st-helens-small.las file into pgpointcloud.
