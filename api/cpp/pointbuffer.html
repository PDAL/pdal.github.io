class pdal::PointBuffer

A PointBuffer is the object that is passed through pdal::Stage instances to form a pipeline.

A PointBuffer is composed of a pdal::Schema that determines the layout of the data contained within, along with a dictionary of pdal::Metadata entries. The capacity of a PointBuffer is determined by the number of points it contains, and the number of points possible in a PointBuffer is limited to std::numeric_limits<boost::uint32_t>::max(). Underneath the covers, a PointBuffer is simply the composed array of bytes described in the pdal::Schema. You can operate on the raw bytes if you need to, but PointBuffer provides a number of convienence methods to make things easier.

Note

The arrangement of PointBuffer’s bytes might either be point-interleaved or dimension-interleaved, with point-interleave being the default organization. If you are directly modifying a PointBuffer’s bytes, you must respect the pdal::pointbuffer::Orientation.

void setRawField(Dimension const & dim, boost::uint32_t pointIndex, const void * value)

set the value for a given :cpp:class:pdal::Dimension dim at the given point index, inferring the size of the value from the byteSize of the dimension.

Parameters
  • dim -

    The dimension to select.

  • pointIndex -

    The point index of the PointBuffer to select.

  • value -

    The value to set, which must have a width of at least dim.byteSize().

    Warning

    This operation performs a memcpy from input parameter ‘value’ of length dim.getByteSize(), so ‘value’ must be guaranteed by the caller to be at least as long as the dimension’s byte size. This operation will hammer the contents of the pdal::Dimension with the contents of parameter ‘value’, so their underlying types should be ensured to match via an accurate schema.
