Use Cases¶

This document lists some use cases that we would like PDAL to be able to satisfy. We are speaking here about PDAL at the API level, and not the command line apps that come with it (although in many cases the apps will directly address these use cases).

Note we have the cases explicitly numbered, so we can refer back to them from elsewhere.

Many of these cases these use cases are composable (such as 1a and 3a).

This list is intended to be representative of workflows we want to target, but is by no means intended to be an exhaustive list.

The Use Cases¶

  1. straight format conversion
    1. convert an ASCII file to LAS
    2. compress LAS to lossless LAZ
  2. mosaicking and demosaicking
    1. user has a set of N tiles and wants to merge them into 1 file
    2. user has 1 large file and wants to chop up into N files, based on a given bbox size or a given target file size (or point count)
  3. basic filtering/processing operations
    1. crop a file to a given spatial extent
    2. remove points with a given classification
    3. decimate the file (discard 9 of every 10 points)
    4. ...
  4. info and metadata
    1. display (or dump to XML) the properties of the point cloud (extents, number of points, etc)
    2. display/dump the spatial reference system of the file
    3. display/dump any metadata in the file, such as creation date, weather conditions, etc
    4. collect and display statistics about the file, such as the elevation distribution, point density, etc
  5. projection
    1. assign an SRS to a file that does not have one (or overwrite the existing one)
    2. reproject the points in a file to a new SRS
  6. indexing and caching
    1. construct a spatial index for a point cloud, such that points within a relatively small given region can quickly be accessed
    2. merge (mosaic) two files that are both indexed
    3. given a file, allow for repeated access to a point during a single run without additional I/O
  7. DEM construction, rasterization, colorization
    1. given an XYZ point cloud, produce a regular XY grid of the Z elevation points
    2. similar to 7a, but colorize each point based on a given color ramp and the elevation height Z
  8. 3D visualization
    1. basic, static display of a point cloud
    2. same as (a), but with colorization of Z, simple filtering, etc
    3. use indexing and caching (6a and 6c) to provide useful interactive behaviour
    4. given a DEM and a raster, display the raster draped over the surface
  9. Fixup, Cleanup
    1. given a file with an incorrect header (such as incorrect extents), create a file with the header corrected by otherwise identical
