Printing a Single Point

Purpose:
  • Learn how to run PDAL via Docker
  • Verify PDAL is working correctly
  • Learn about point cloud data types

Workshop Materials

Command (first point)

In your Docker Quickstart Terminal, issue the following:

1
2
docker run -v /c/Users/Howard/PDAL:/data -t pdal/pdal \
    pdal info /data/exercises/info/interesting.las -p 0
  1. docker: All our commands start with docker
  2. run: Tells docker we're going to run an image
  3. -v /c/Users/Howard/PDAL:/data: Maps our workshop directory to a directory called /data inside the container.

Command (first point)

1
2
docker run -v /c/Users/Howard/PDAL:/data -t pdal/pdal \
    pdal info /data/exercises/info/interesting.las -p 0
  1. pdal/pdal: Run the image with name pdal/pdal
  2. pdal: Inside the pdal/pdal image, run the pdal command
  3. info: info is a "kernel" in PDAL-speak. It is a unit of functionality driven by the command line.
